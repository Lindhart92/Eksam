    Cascading Style Sheets

    Css står for Cascading Style Sheets. Simpelt sagt er Css det der bestemmer en hjemmesides udseendet og opstilling og derved pæn og ordentlig at se på, uden Css ville html elementerne bare have deres standard udseendet samt placeringer. Css gør intet alene, kun hvis det er knyttet til enten elementer, #id eller .class i en html fil. Forskellige metoder til at linke sit CSS. Inline CSS skrives direkte ind i et html element ligesom attributes. (header style=”color:black;”) Inline CSS er ikke en praktisk metode da den ikke kan gælde for flere elementer men kun for det tag som den er skrevet ind i. Internal CSS er også skrevet direkte ind i en html fil men i stedet for at skrive det ind i et specifikt tag så bliver det skrevet i "head" og kan derfor bruges til flere elementer i html filen. Internal CSS kan dog kun bruges i det html den er skrevet i. External CSS er den mest brugte metode når man arbejder med CSS, man skriver ikke noget i sin html fil men i stedet laver man en separat fil som man kalder et stylesheet. Med et stylesheet kan man bruge den samme CSS til flere html filer og derved gøre det lettere at ændre udseendet af mange side på en gang. .

    Css regler består af to elementer. Den første del af en CSS regl er selectoren som bestemmer hvad reglen skal gælde for. Det er normalt en .class eller et #id som man linker reglen til. Derfor er class og id vigtige i ens html da man kan vælge at linke en regl til flere elementer så man slipper for at lave en ny regl for hvert element eller class.

    Reglens næste del er Declaration og er det der giver reglen noget at lave såsom at ændre font størrelse ”Font-size”. Declaration består af en property og en value. Property fortæller hvad der skal ændres i elementet og value bestemmer hvordan det skal ændres. Det kan f.eks. være ”text-align” som er en property og ”center” som er valuen. p {text-align: center}
